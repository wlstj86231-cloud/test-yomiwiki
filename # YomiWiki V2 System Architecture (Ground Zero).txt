# YomiWiki V3 Unified Architecture

**타깃:** 미국 오컬트 매니아 (영문판)
**목표:** 나무위키 아키텍처 기반 + 기밀문서 보관소(Clinical Mode) 다크 테마 위키 커뮤니티

## 1. 프로젝트 청사진 (Project Blueprint)
인프라 구축과 라우팅 안정화는 완료되었다. 이제부터는 나무위키 수준의 '정보 밀도'와 '커뮤니티 상호작용'을 구현하는 데 집중한다.
* **Phase 1 (완료):** CF Pages 배포, D1 DB 연동, SPA 라우팅 및 다크 테마 UI 기초 공사.
* **Phase 2 (위키 엔진 고도화 - 현재 단계):** 자체 위키 파서(문단, 목차, 각주) 및 문서 리비전 시스템 구축.
* **Phase 3 (커뮤니티 통합):** 단일 문서 하단 댓글 연동, 유저 기여도 및 등급 시스템.
* **Phase 4 (특수 포맷팅):** 오컬트 특화 인포박스(정보상자), 미디어(이미지) 업로드 및 카테고리 분류.

## 2. 맥락노트 (Contextual Notes)
* **[2026-03-19] 파서 고도화 및 자동화 프로세스 구축:** Parser V2(TOC/스크롤 앵커링) 및 V3(각주) 정밀 재구현 중. 작업 완료 후 깃 자동 푸시 및 실시간 사이트(yomiwiki.com) 반영 확인 프로세스 수립.
* **디자인 원칙 (Design Preservation Mandate):** 
    * **[최우선 사항]** 현재의 검은색/주황색 기반 '기밀문서(Clinical Mode)' 디자인 톤앤매너를 절대적으로 유지한다. 
    * 사용자가 현재 디자인에 매우 만족하고 있으므로, 어떠한 기능 추가나 수정 시에도 레이아웃, 색상, 폰트, interactive feedback 등의 시각적 요소는 현재의 상태를 보존해야 한다. 
    * 디자인 변경이 필요한 경우 반드시 사용자의 명시적 허가를 받아야 한다.

## 3. 핵심 작업 체크리스트 (Action Items)

1v. 현재 프로젝트의 모든 파일과 폴더를 압축해서 백업본을 하나 만들어줘.
2v. 문서 데이터들이 저장되어 있는 데이터베이스(또는 파일) 코드를 열어줘.
3v. 모든 문서 데이터에 1번부터 순서대로 안 겹치는 고유 번호(ID) 항목을 새로 추가해줘.
4v. 댓글 데이터들이 저장되어 있는 데이터베이스(또는 파일) 코드를 열어줘.
5v. 모든 댓글 데이터에 '이 댓글이 어떤 문서 번호(ID)에 달린 건지'를 적어둘 빈칸 항목을 새로 추가해줘.
6v. 웹사이트의 주소(URL) 이동을 관리하는 라우터(Router) 코드를 열어줘.
7v. `yomiwiki.com/w/문서번호` 형태로 접속할 수 있는 새로운 주소 규칙을 하나 만들어줘.
8v. 사용자가 저 주소로 들어오면, 여러 개가 보이는 목록이 아니라 '딱 하나의 문서만 보여주는 화면(Detail Page)'으로 이동하게 연결해줘.
9v. 기존에 문서들이 쭉 나열되어 있던 게시판 화면 코드를 열어줘.
10v. 목록을 클릭했을 때 글이 아래로 스르륵 열리는 기능(아코디언 UI) 코드를 찾아서 완전히 삭제해줘.
11v. 이제 게시판 목록에는 문서의 '제목'만 화면에 보이도록 코드를 수정해줘.
12v. 게시판 목록에서는 문서의 '본문 내용' 데이터를 아예 불러오지 않도록 코드를 지워줘. (속도를 빠르게 하기 위함)
13v. 게시판 목록에 있는 '문서 제목'에 마우스를 올리면 누를 수 있는 버튼(링크)으로 작동하게 만들어줘.
14v. 그 '문서 제목' 버튼을 누르면, 아까 7번에서 만든 `yomiwiki.com/w/해당문서번호` 주소로 페이지가 넘어가도록 코드를 짜줘.
15v. '단일 문서 전용 화면'으로 쓸 새로운 HTML(또는 컴포넌트) 빈 파일을 하나 만들어줘.
16v. 이 빈 파일에 코드를 짤 건데, 가장 먼저 사용자가 접속한 인터넷 주소창에서 '문서 번호'를 숫자로 읽어오게 해줘.
17v. 읽어온 '문서 번호'를 가지고, 데이터베이스에 가서 그 번호와 똑같은 문서 데이터 딱 1개만 찾아와줘.
18v. 화면 맨 위쪽에 찾아온 데이터의 '문서 제목'을 아주 큰 글씨로 띄워줘.
19v. 제목 바로 아래쪽에는 찾아온 데이터의 '문서 본문 내용'을 화면에 띄워줘.
20v. 본문 글씨가 화면 옆으로 튀어나가지 않게, 화면 끝에 닿으면 자동으로 줄이 바뀌도록 디자인(CSS)을 설정해줘.
22v. 여기에 들어갈 '문서 전용 댓글 창' 빈 파일을 하나 새로 만들어줘.
23v. 이 댓글 창 코드에서, 현재 화면이 띄워주고 있는 '문서 번호'를 확인하게 해줘.
24v. 댓글 데이터베이스에 가서, 수많은 댓글 중 아까 확인한 '문서 번호'와 똑같은 번호가 적힌 댓글들만 전부 골라서 가져와줘.
25v. 가져온 댓글들을 단일 문서 화면 맨 아래 공간에 시간 순서대로 띄워줘.
26v. 띄워진 댓글들 맨 아래에, 사용자가 글씨를 쓸 수 있는 네모난 '입력창'을 만들어줘.
27v. 입력창 바로 옆에 누를 수 있는 '등록' 버튼을 만들어줘.
28v. '등록' 버튼을 누르면, 사용자가 입력창에 쓴 글씨를 새로운 댓글 데이터로 만들어서 데이터베이스에 저장해줘.
29v. (중요) 저장할 때, 이 댓글이 지금 열려있는 페이지의 '문서 번호'를 반드시 세트로 같이 기억해서 데이터베이스에 넣게 해줘.
30v. 저장이 성공하면, 방금 쓴 댓글이 화면에 즉시 나타나도록 화면 아랫부분만 새로고침 해줘.
31v. 단일 문서 화면의 '문서 제목'과 '문서 본문' 사이에 구글 애드센스 광고 코드를 넣을 수 있는 빈 상자(div)를 하나 만들어줘.
32v. 단일 문서 화면의 '문서 본문'이 끝나고 '댓글 창'이 시작되기 직전에도 애드센스 광고 빈 상자를 하나 만들어줘.
33v. 구글 검색 봇이 이 문서를 잘 찾을 수 있게, 이 화면의 `<head>` 태그 안의 `<title>` 부분에 '문서 제목'이 자동으로 들어가게 코드를 짜줘.
34v. 단일 문서 화면의 본문 코드에서, 제목 역할을 하는 큰 글씨들(h2, h3 태그)을 모두 찾게 해줘.
35v. 찾아낸 큰 글씨들의 텍스트만 뽑아서 리스트(배열)로 따로 저장해줘.
36v. 단일 문서 화면의 본문이 시작되는 맨 윗부분에 빈 네모 상자를 하나 만들어줘.
37v. 그 네모 상자 안에 방금 저장한 텍스트 리스트를 위에서 아래로 순서대로 띄워줘.
38v. 상자 안에 띄운 텍스트들 앞에 '1.', '1.1.', '2.' 처럼 나무위키식 번호를 자동으로 매겨줘.
39v. 목차 상자 안의 텍스트를 마우스로 누르면, 본문에 있는 해당 글씨 위치로 화면이 스크롤되어 내려가게 연결해줘.
40v. 문서 본문 데이터를 화면에 띄우기 직전에, 글씨 중에 대괄호 두 개 `[[` 와 `]]` 로 감싸진 단어가 있는지 찾는 규칙을 만들어줘.
41v. 그런 단어를 찾으면 대괄호를 지우고, 그 단어를 누를 수 있는 파란색 글씨(링크)로 바꿔줘.
42v. 그 파란색 글씨를 누르면 `yomiwiki.com/w/그단어` 주소로 페이지가 이동하게 코드를 짜줘.
43v. 만약 `[[` `]]` 안에 적힌 단어와 똑같은 제목의 문서가 데이터베이스에 존재하지 않으면, 파란색 대신 빨간색 글씨로 보여주게 규칙을 추가해줘.
44v. 데이터베이스에 '문서 역사'들을 따로 모아둘 새로운 저장소(테이블)를 하나 만들어줘.
45v. 사용자가 문서 내용을 수정하고 저장할 때마다, 수정하기 직전의 옛날 내용들을 '문서 역사' 저장소에 복사해서 넣어줘.
46v. 복사할 때 '누가 고쳤는지(IP 주소나 아이디)'를 파악해서 빈칸에 같이 적어줘.
47v. 복사할 때 '언제 고쳤는지(날짜와 시간)'를 파악해서 빈칸에 같이 적어줘.
48v. 단일 문서 화면 맨 위쪽에 '역사'라는 이름의 버튼을 하나 만들어줘.
49v. '역사' 버튼을 누르면 이 문서의 과거 기록들이 시간 순서대로 나열되어 보이는 새로운 화면 페이지를 만들어줘.
50v. 본문 내용 중에 `[*` 와 `]` 로 감싸진 글씨가 있는지 찾는 규칙을 만들어줘.
51v. 찾으면 감싸진 안쪽 글씨들을 본문 화면에서 보이지 않게 숨겨줘.
52. 숨긴 원래 글씨 자리에는 위첨자로 파란색 작은 숫자 `[1]`, `[2]` 를 순서대로 달아줘.
53. 단일 문서 화면 본문 맨 아래(댓글 창 위)에 가로선을 하나 긋고, 아까 숨겼던 글씨들을 모아서 보여줄 '각주 공간'을 만들어줘.
54. 본문에 있는 작은 숫자 `[1]`을 누르면, 화면 맨 아래 각주 공간에 있는 1번 설명으로 화면이 이동하게 연결해줘.
55. 웹사이트의 오른쪽(또는 왼쪽) 빈 공간에 항상 떠 있는 길쭉한 빈 상자(사이드바)를 만들어줘.
56. 데이터베이스에서 가장 최근에 수정되었거나 새로 만들어진 문서 10개를 시간 순서대로 찾아와줘.
57. 찾아온 10개 문서의 제목을 사이드바 빈 상자 안에 위에서부터 아래로 띄워줘.
58. 제목 바로 옆에 언제 바뀌었는지 시간도 아주 작게 적어줘.
59. 사이드바에 있는 제목을 누르면 그 문서 화면으로 바로 이동하게 링크를 걸어줘.
60. 데이터베이스의 문서 데이터에 '이 문서는 어떤 분류에 속하는지' 적을 수 있는 빈칸 항목을 새로 만들어줘.
61. 단일 문서 화면 맨 위(제목보다 위)에 네모난 가로 띠를 하나 만들어줘.
62. 그 띠 안에 데이터베이스에서 가져온 이 문서의 '분류' 이름을 띄워줘.
63. '분류' 이름을 누르면, 똑같은 분류를 가진 다른 문서들의 목록만 쭉 모아서 보여주는 새로운 화면 페이지를 만들어줘.
64. 웹사이트 맨 위쪽(헤더)에 검색어를 입력할 수 있는 하얀색 입력창을 만들어줘.
65. 검색 입력창 옆에 돋보기 모양의 '검색' 버튼을 만들어줘.
66. 입력창에 글씨를 치면, 데이터베이스 문서 제목들 중에서 그 글씨가 포함된 것을 실시간으로 찾게 해줘.
67. 글씨를 치는 동안 입력창 바로 아래에 찾아낸 문서 제목들을 미리보기 리스트로 띄워줘.
68. 돋보기 버튼을 누르면, 제목뿐만 아니라 본문 내용에 그 글씨가 포함된 모든 문서를 찾아서 '검색 결과 화면'에 띄워줘.
69. 단일 문서 화면의 본문이 끝나고 각주가 시작되기 직전에, '이 문서와 같이 보기 좋은 문서' 목록을 보여줄 네모난 공간을 만들어줘.
70. 현재 보고 있는 문서와 똑같은 '분류'를 가진 다른 문서 3개를 데이터베이스에서 무작위(랜덤)로 뽑아와줘.
71. 뽑아온 3개 문서의 제목을 아까 만든 네모난 공간에 띄우고, 누르면 그 문서로 이동하게 연결해줘.
72. 구글 애드센스의 '디스플레이 광고' 코드를 복사해서 단일 문서 화면의 사이드바 맨 밑부분에 항상 떠있게 붙여넣어줘.
