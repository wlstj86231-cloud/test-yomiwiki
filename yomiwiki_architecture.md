## 1. 계획서 (Project Blueprint)
코드가 엉키지 않도록 프론트엔드와 백엔드를 철저히 분리하여 단계별로 개발한다.
* **1단계 (기초 공사):** 폴더 구조(public/functions) 분리 및 정적 뼈대 구축
* **2단계 (데이터/통신):** D1 데이터베이스 스키마 설계 및 CRUD API 연동
* **3단계 (위키 엔진):** 자체 위키 파서, 리비전(역사), 토론 시스템 구축
* **4단계 (고도화/보안):** 아래 100가지 체크리스트에 기반한 기술적 결함 해결 및 최적화

## 2. 맥락노트 (Contextual Notes)
* **Zero Base 결정 이유:** 기존 코드는 UI 렌더링과 데이터 통신, 라우팅이 난잡하게 섞여 치명적 문법 에러(`SyntaxError`) 발생 시 전체 화면이 붕괴하는 결함이 있었음.
* **관심사 분리(Separation of Concerns):** HTML/CSS는 시각적 렌더링, 자바스크립트는 데이터 통신, 서버 로직(Workers/Pages)은 DB 관리만 전담하도록 아키텍처를 강제함.
* **[2026-03-18] 긴급 복구 내역:**
    1. **share-modal.js 마비 해결:** DOM 요소 부재 시 스크립트 중단을 막는 Null Check 방어 로직 도입 및 `DOMContentLoaded` 래핑.
    2. **API 500 에러 해결:** D1 바인딩 체크 및 제목 정규화(공백/언더바 혼용) 호환성 확보. 상세 에러 JSON 반환 로직 강화.
    3. **UI 붕괴 해결:** SSR 환경에서 CSS 경로가 깨지던 현상을 절대 경로(`/style.css`)로 수정하고 글로벌 리셋 코드 삽입.
